# 1. Introduction

InsiderGuard AI is an Insider Threat Detection and Digital Forensics platform designed for Windows environments.

The platform continuously collects Windows Security Events, analyzes user activities, detects suspicious behavior, generates security alerts, preserves forensic evidence using blockchain integrity verification, and assists analysts through AI-generated investigation reports.

The primary users of InsiderGuard AI are:

- Security Operations Center (SOC) Analysts
- Digital Forensics Investigators
- Incident Response Teams
- Security Administrators

# 3. Dashboard

The Dashboard provides an overview of the security status of the organization.

Displayed information includes:

• Total Login Events

Number of successful authentication events received from monitored Windows hosts.

• Alerts

Number of generated security alerts.

Alerts represent suspicious activities detected by the Risk Assessment Engine.

• Open Incidents

Incidents that require analyst investigation.

Only unresolved incidents are counted.

• High-Risk Users

Number of users whose risk score exceeds the configured threshold.

• Evidence

Total preserved forensic evidence.

# 5. Incident Center

The Incident Center displays all security incidents generated by the detection engine.

Each incident contains:

Incident ID

Affected User

Severity

Status

Description

Creation Time

---

## Incident Status

Available statuses:

OPEN

The incident has not yet been reviewed.

IN_PROGRESS

An analyst is currently investigating.

CONTAINED

Containment actions have been applied.

Example:

Account disabled

Host isolated

Password reset

RESOLVED

Investigation completed successfully.

FALSE_POSITIVE

The incident was determined to be benign.

---

## Updating Status

Open an Incident.

Locate the Status field.

Select the new status from the dropdown list.

The update is saved automatically.

# 6. Evidence Center

Evidence represents preserved forensic snapshots.

Each evidence item contains:

Evidence ID

Incident ID

Username

Evidence Type

SHA256 Hash

Creation Time

---

## Evidence Detail

Each evidence includes:

General Information

Evidence Snapshot

Related Incident

Blockchain Record

Integrity Verification

---

## Snapshot

Snapshot contains captured investigation data.

Example:

Username

Hostname

IP Address

Risk Score

Alert Information

Collected Metadata

---

## SHA256

Every evidence snapshot is hashed using SHA256.

The hash is used for integrity verification.

---

## Blockchain

Each evidence is stored in a blockchain-inspired ledger.

Displayed information includes:

Block Index

Evidence Hash

Previous Hash

Current Block Hash

Nonce

Creation Time

---

## Integrity Verification

If

Evidence Hash == Blockchain Hash

Integrity Status

VALID

Otherwise

Integrity Status

FAILED

This ensures evidence has not been modified.

# 8. Blockchain Verification

Blockchain verification protects forensic evidence from tampering.

Each block contains:

Evidence Hash

Previous Block Hash

Current Block Hash

---

## Chain Verification

The backend validates:

Hash correctness

Chain linkage

Evidence integrity

---

## Verification Results

VALID

All blocks are consistent.

INVALID

Evidence or blockchain may have been modified.

Immediate investigation is recommended.

# 10. Best Practices

Always verify evidence integrity before relying on evidence.

Review Windows Events together with AI analysis.

Do not rely solely on AI-generated recommendations.

Update incident status throughout the investigation process.

Document remediation actions.

Preserve evidence before making changes to the affected system.

# 11. Troubleshooting

Dashboard shows no data

Possible causes:

Collector not running

Backend unavailable

Database connection failed

No events received

---

Event Explorer is empty

Verify that:

Collector is connected

Windows Security Logs are being forwarded

Backend API is running

---

Cannot generate AI Investigation

Possible causes:

Incident not found

Evidence missing

Database unavailable

Backend error

---

Blockchain verification failed

Possible causes:

Evidence modified

Database inconsistency

Blockchain corruption

Further forensic investigation is required.
